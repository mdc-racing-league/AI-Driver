## 2. Run A — Elevation Calibration (Control Run)

**Purpose:** Collect z-field (altitude) data to determine whether the s08 kink
at ~1950 m is a terrain crest (elevation drop → grip loss) or a geometric
corner. No segment changes — this is a clean control lap.

**Segments file:** `telemetry/segments.yaml` (no changes)

### Window A — confirm TORCS is ready, then watch the lap live

### Window B — launch the run
```powershell
python src\driver_baseline.py --segments telemetry\segments.yaml --notes "Run A elev-cal s08=98 baseline z-collection"
```

### Post-lap analysis (Window B)
Run these four commands in order:

```powershell
# 1. Validate the run and capture the RUN_FOLDER name
python scripts\validate_run.py

# 2. Find any off-track events
python scripts\find_offtracks.py

# 3. Segment performance report
python scripts\segment_report.py

# 4. Elevation profile — zoom to the kink window
#    Replace <RUN_FOLDER> with the folder name shown by validate_run.py
python scripts\elevation_profile.py telemetry\runs\<RUN_FOLDER> --zoom 1850 2100
```

### Decision Gate — Run A → Run B

Read the elevation profile output for the 1850–2100 m window:

| Finding | Interpretation | Run B action |
|---|---|---|
| z drops **> 1 m** across 1850–2100 m | Crest confirmed — grip loss at top | Use `segments_run_b.yaml` as written (92 km/h micro-cap at 1930–1970 m) |
| z is **flat** (< 0.5 m variation) | Geometric corner, not a crest | Edit `telemetry/segments_run_b.yaml`: change `s08b_kink` target to **88 km/h** and narrow range to 1940–1960 m for a tighter turn-in cap |
